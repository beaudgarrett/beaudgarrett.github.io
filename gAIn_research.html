gAIn: Neural Regression for Synthetic Hypertrophy Response Estimation, Volume Optimization, and Interactive Strength-Training Decision Support

Beau Garrett

Independent Research — beaugarrett.dev

April 2026

Abstract

We present gAIn, a research-oriented machine learning system for estimating resistance-training hypertrophy potential from program design and recovery variables. The final pipeline trains four regressors on a 2,400-sample synthetic cohort spanning volume, intensity, frequency, recovery, nutrition, and exercise-selection covariates, then deploys the best model inside an interactive browser application for live program feedback. Among linear regression, random forest, gradient boosting, and a feed-forward neural network, the neural model achieved the strongest held-out performance with RMSE 12.7848, MAE 9.2070, and R2 0.8549, outperforming the heuristic baseline by 45.84% on RMSE and 53.09% on MAE. We further report permutation-based feature importance, score-distribution structure, convergence behavior, and calibration against a rule-based benchmark. The present implementation uses a multilayer perceptron rather than a transformer, reflecting the low-dimensional tabular nature of the deployed MVP; however, the results suggest that neural nonlinear function approximation materially improves ranking and sensitivity over hand-built heuristics alone.

Keywords: hypertrophy modeling, neural regression, resistance training, synthetic dataset generation, permutation importance, interactive decision support, heuristic benchmarking, MLPRegressor

1. Introduction

Practical hypertrophy planning is governed by interacting variables: weekly set volume, repetition ranges, proximity to failure, recovery capacity, training age, and nutritional context. In human coaching practice these factors are often synthesized heuristically, producing advice that is interpretable but coarse. gAIn investigates whether a machine learning system can map the same variables into a more sensitive and better calibrated hypertrophy score while remaining transparent enough for product use.

The central research question is whether nonlinear regressors, and especially a small feed-forward neural network, can outperform a deliberately interpretable heuristic baseline on a structured synthetic training domain. This framing is appropriate for an MVP because real hypertrophy labels are scarce, confounded, and difficult to standardize, whereas a synthetic benchmark enables controlled evaluation of architecture choice, sensitivity, and user-facing decision support. The resulting application exposes the learned response surface through live prediction, program warnings, and volume optimization visualizations.

Unlike sequence-heavy domains that naturally favor transformers, the deployed gAIn input is a compact tabular state vector. Accordingly, the strongest neural result came from a multilayer perceptron rather than a transformer. This distinction matters: the paper documents the neural architecture that was actually used, benchmarks it against strong tree ensembles, and reserves transformer-style extensions for future longitudinal training-log versions of the system.

2. Methodology

2.1 Synthetic Cohort Construction

The synthetic cohort contains 2,400 program instances across six muscle groups, three exercise-composition categories, and three calorie states. Each observation includes weekly sets, reps per set, average rep range, frequency per week, average RPE, rest time, body weight, training age, sleep, and protein intake. Muscle-group-specific optimal set ranges are encoded so that the target function rewards productive overload while penalizing both undertraining and junk volume.

The target label is a bounded continuous hypertrophy score generated from a latent physiological signal and then squashed into a 1–99 scale. This design intentionally produces nonlinear interactions between overload, recovery, and nutritional state rather than a trivial linear mapping.

z = 54 - volume_penalty + frequency_bonus + reps_bonus + intensity_bonus + rest_bonus + sleep_bonus + protein_bonus + calorie_bonus + exercise_bonus + training_age_terms - undertraining - junk_volume + recovery_interaction + overload_synergy + ε

score = clamp(100 / (1 + exp(-(z - 58) / 11.5)) + εfinish, 1, 99)

2.2 Model Family

Four supervised regressors were trained on the same held-out split: linear regression, random forest, gradient boosting, and a feed-forward neural network. A heuristic baseline was retained as a non-ML comparator. The linear model functions as a low-capacity control, the tree ensembles capture nonlinear interactions without feature scaling assumptions, and the neural network tests whether a compact learned nonlinear manifold can better approximate the synthetic target function.

  • Linear regression: standardized baseline with limited nonlinear capacity.
  • Random forest: 400 trees, min_samples_leaf=2.
  • Gradient boosting: default boosting ensemble with additive nonlinear learners.
  • Neural network: StandardScaler -> MLPRegressor(192, 96, 48) with ReLU activations, Adam optimization, L2 regularization, and early stopping.
Architectural note. No transformer was used in the current production build. The deployed neural model is a feed-forward multilayer perceptron because the MVP operates on a single tabular snapshot rather than a temporal sequence. A transformer becomes more appropriate once longitudinal athlete history, mesocycle sequences, or exercise-level time series are introduced.

2.4 Training Protocol and Evaluation

The regression benchmark uses a fixed random seed and a 75/25 train-test split. All predictions are clipped into the valid 0–100 interval at inference time. The neural network uses internal early stopping with a 15% validation fraction drawn from the training partition. Model quality is reported with MAE, RMSE, and R2, with lower error and higher explained variance preferred. In addition, a permutation-importance pass is executed on the held-out set to recover interpretable post hoc relevance estimates for the best model.

3. Quantitative Results

The feed-forward neural network was the strongest model across all tracked metrics. It outperformed random forest by 10.70% on RMSE and 14.89% on MAE, and exceeded gradient boosting by 11.97% on RMSE and 15.37% on MAE. Relative to the rule-based heuristic baseline, the neural model reduced RMSE from 23.6051 to 12.7848 and increased explained variance from 0.5055 to 0.8549.

5. Discussion

The benchmark establishes three central findings. First, a feed-forward neural network is meaningfully useful in this domain: it is not merely competitive with the tree models, but measurably superior on all primary regression metrics. Second, the heuristic baseline remains directionally informative but leaves substantial performance on the table, especially in the upper half of the score range where many practically relevant programs live. Third, the learned function is interpretable enough for a product surface because post hoc importance analysis recovers plausible drivers rather than opaque noise.

The feature-importance profile is especially instructive. weekly_sets alone contributes 19.59% of the measured permutation signal, and avg_rep_range contributes 14.98%, reflecting the centrality of dose and execution context in the synthetic objective. Muscle-group indicators collectively remain important because the target generator encodes muscle-specific optimal set windows. Sleep and rep structure also remain visible, supporting the idea that the model is learning a coupled training-recovery surface rather than a single-axis volume heuristic.

It is also important to interpret the absolute numbers honestly. These are strong results for a synthetic MVP, not proof of biological truth. Because the labels are generated from a designed scoring function, the neural network is best understood as an effective approximator of that structured latent process. This is still valuable: the app now behaves more sensitively, benchmark quality is quantifiable, and the decision-support surface can be explored interactively. But scientific validity against real hypertrophy outcomes would require external longitudinal data, uncertainty analysis, and athlete-level calibration.

On the question of transformers, the present results argue against unnecessary architectural inflation at this stage. The problem solved here is tabular snapshot regression, and the MLP already reaches R2 0.8549. A transformer would be justified only if the product were upgraded to ingest sequential workout histories, exercise ordering, progression dynamics, fatigue trajectories, or wearable-derived recovery time series. In other words, transformer capacity should follow sequence structure, not precede it.

6. Conclusion

gAIn demonstrates that a small but well-chosen neural regressor can materially improve hypertrophy-score estimation over both linear models and hand-written heuristics in a synthetic research setting. The deployed MLP achieved the best held-out metrics in the benchmark, reduced baseline RMSE by 45.84%, and provided sufficiently interpretable structure through permutation importance to support a polished interactive product. The resulting system now combines synthetic physiology-inspired labeling, model comparison, neural prediction, graphical analysis, and user-facing volume optimization in one coherent workflow.

The most credible next step is not a larger static model, but richer data. A future version should incorporate longitudinal mesocycle sequences and observed progression trajectories, at which point transformer encoders or temporal neural architectures could be evaluated legitimately rather than rhetorically. Until then, the current feed-forward neural approach is the correct description of what was built and the strongest model actually used in the gAIn MVP.
